Realtime Voxel-Based Path-Tracer

The latest project uses 3D textures to save lighting data per voxel, allowing us to render an environment to high fidelity. Moving trough the environment then only needs minimal updates on the edges of the lighting textures and changes in the environment, enabling high framerates with interesting lighting.

Paths are traced for the screen-render, light sources and scattering light seperately. Source rays travel in all directions from a unidirectional light source, optimised in preserving thread locality. Voxels contain a crude probability density function stored in 32 bits per voxel, allowing for more precision by sampling better regions. This image shows a room primarily lit by scattered light trough a small window and door opening.

Rays from light sources get bounced around the scene, allowing for reflections of light sources appearing trough mirrored surfaces and refracting trough glass. This image shows 'sunlight' hitting a stained glass painting, projecting the image onto the floor, and bouncing trough the mirror strips to the walls. Because of the separate scattering rays, this light can then be scattered to also light up the room according to the colorful patterns.

These source rays also refract on entering different materials. This image shows total internal reflection occuring in the colored glass, causing the light rays to converge again in the center. Again, this light is saved in the texture and is then available for the scatter to bounce around.

Other projects

First Renderer

[2018]

Wanting to create a 3D physics engine, a 3D rendering engine was necesary. This project uses a basic pipeline of converting points from 3D space to screenspace, and rendering these in triangle meshes.
Spheres were rendered by solving an ellipse formula on 5 points of the outline of the sphere in screenspace. These points were generated by solving for lines tangent to the sphere and trough the camera position.

Black Hole CPU render

[April 2021]

This program renders an image using marching rays. These rays change direction each step based on the 'gravitational pull', and intersect with the rings, stars, or fall into the black holes.
Made in the span of an afternoon

First GPU Ray-Tracer

[September 2021]

After figuring out a trick using a shader applied to an image in the programming app I used, encoding 3d environments to pixel data and decoding it in glsl allowed for a rudamentary gpu raytrace pipeline.
First ray-tracer to run consistent at 60fps and high resolution.
Post-processing was not possible in this pipeline, so bloom was achieved trough a ray-object distance calculation.

First Voxel-Based Ray-Tracer

[November 2021]

Basic tracing, but uses octree traversal for ray-object intersection.
Started playing with post-processing effects

First Full Path-Tracer

[August 2022]

Full path tracing, with stochastic rays for scattering. Also has some features to try to improve performance, such as sampling the image for light-data when a scatter ray stays in screenspace, and sampling the previous frame for each pixel for what I later learned is temporal accumulation. Worked well realtime but still a bit noisy.
